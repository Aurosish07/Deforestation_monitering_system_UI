# Deforestation Monitoring Map – Technical Deep Dive

This document explains the exact mapping mechanics, external APIs, and geometric rendering processes behind the deforestation monitoring interface.

---

## 1. The Core Mapping Engine
The application utilizes an open-source javascript library called **Leaflet.js**, wrapped for React via `react-leaflet`. Leaflet handles the heavy math required to convert pixel coordinates on your screen into real-world geographic coordinates (Latitude / Longitude).

When the map boots up, it instantiates an interactive `<MapContainer>` set to a global EPSG:3857 coordinate reference system (Standard Web Mercator).

---

## 2. Satellite Imagery API (The Basemap)
The photorealistic satellite foundation you see is completely generated by **Esri's World Imagery API**.
- **Endpoint**: `https://server.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer/tile/{z}/{y}/{x}`
- **How it works**: Leaflet calculates exactly which part of the globe you are looking at based on your zoom level (`{z}`) and xy-grid position (`{x}, {y}`). It fires hundreds of asynchronous GET requests to Esri's servers to fetch these specific 256x256 pixel image "tiles," stitching them together seamlessly to create a continuous globe.

---

## 3. Drawing the Bounds (Area Selection)
To analyze forest degradation, the user needs to define a search parameter. This is handled by a specialized extension called **`leaflet-draw`**.

### The Flow:
1. The user clicks the rectangular draw tool and drags across the imagery.
2. `leaflet-draw` translates this pixel drag into geographic points.
3. Once the mouse is released, a `draw:created` event fires in the browser.
4. The system intercepts the mathematical bounds of this shape (`layer.getBounds()`) and extracts four critical coordinates:
   - **West** (Minimum Longitude)
   - **South** (Minimum Latitude)
   - **East** (Maximum Longitude)
   - **North** (Maximum Latitude)
5. This extracted array `[W, S, E, N]` is known as a **Bounding Box (bbox)** and represents the precise scanning parameter sent to the backend pipeline.

---

## 4. Submitting the Analysis (API Contract)
When the user clicks "Run Analysis", the front-end sends a `POST` request to the backend with four primary payloads:
- `bbox`: The coordinates derived above.
- `date_t0`: The initial baseline date (Reference Imagery).
- `date_t1`: The target date (Comparison Imagery).
- `threshold`: A sensitivity float (e.g., `0.2`) determining how strictly to compute the NDVI gradient diff.

The backend processes the raw Sentinel-2 GeoTIFF imagery using `rasterio` and `scikit-image`, calculating vegetation loss natively, and finally vectorizing these raster pixels back into distinct coordinate geometries.

---

## 5. Rendering the Polygons (GeoJSON)
The backend responds with the completed analysis in standard **GeoJSON Format**. GeoJSON is a spatial data format that looks like this:
```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "geometry": {
         "type": "Polygon",
         "coordinates": [[[lon1, lat1], [lon2, lat2], ...]]
      },
      "properties": { "severity": "critical", "area_ha": 84.2 }
    }
  ]
}
```

### The Rendering Pipeline:
1. The frontend injects this JSON object into Leaflet's `<GeoJSON>` component.
2. Leaflet mathematically iterates over the `coordinates` array for every feature.
3. It maps each coordinate array to physical pixel locations on your screen.
4. It dynamically draws an SVG `<path>` element for each shape, coloring them dynamically based on configuration (e.g., `color: "red"`, `weight: 2`, `fillOpacity: 0.2`).
5. As you zoom or pan the map, Leaflet mathematically re-projects these SVGs in real-time so they perfectly overlay the underlying Esri satellite tiles.

---

## 6. Mathematical Interactivity (Clicking & Tooltips)
When mapping the GeoJSON layer, Leaflet executes a hook called `onEachFeature`.
- For every single polygon mathematically generated above, the frontend attaches a native browser `onClick` event listener directly to its SVG DOM node.
- If an analyst hovers over the polygon, Leaflet computes the geographical centroid (the exact middle point) and dynamically absolute-positions a tiny `div` tooltip reflecting the severity.
- If the polygon is clicked, the `incident id` bound within its `properties` block is emitted to the React engine, which subsequently slides open the Intelligence Drawer rendering the original Before and After imagery proofs natively on the right-hand panel.
